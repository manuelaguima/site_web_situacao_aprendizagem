Modelagem de Banco de Dados

    A modelagem de banco de dados é o processo de criação de uma estrutura organizada para armazenar, gerenciar e acessar dados em um banco de dados. Essa estrutura define como os dados serão organizados e como as diferentes partes desses dados se relacionam. O principal objetivo da modelagem de banco de dados é representar de forma clara e eficiente as informações de uma organização ou sistema, facilitando sua manipulação e uso futuro.

Como funciona modelagem conceitual

A modelagem conceitual é a fase inicial da modelagem de banco de dados, onde o foco está em representar o que será armazenado no banco de dados de maneira abstrata, sem se preocupar com detalhes técnicos ou de implementação. É a visão de alto nível que descreve entidades e relacionamentos, visando capturar a essência dos dados e suas interações.

Principais características

  • Define entidades (ex.: Cliente, Produto, Pedido) e seus relacionamentos (ex.: um Cliente faz vários Pedidos).
  • Não inclui detalhes como tipos de dados ou estrutura de tabelas.
  • É independente de qualquer tecnologia de banco de dados específica.

Objetivo: Fornecer uma visão clara dos dados e suas interações no contexto do negócio, ajudando a entender os principais elementos e suas associações.

Como funciona a modelagem lógica

A modelagem lógica é mais detalhada que a conceitual e começa a definir como os dados serão organizados, mantendo-se ainda independente do sistema de gerenciamento de banco de dados (SGBD) que será usado. Ela define com mais precisão entidades, atributos, chaves primárias, chaves estrangeiras e relacionamentos.

Principais características

  • As entidades são transformadas em tabelas.
  • Inclui os atributos (ou colunas) de cada entidade.
  • Inclui os atributos (ou colunas) de cada entidade.
  • Estabelece chaves primárias e chaves estrangeiras.
  • Ainda não inclui detalhes de implementação específicos, como tipos de dados.

Objetivo: Proporcionar uma visão estruturada dos dados e garantir a integridade e consistência entre tabelas, preparando o terreno para a fase física.

Como funciona a modelagem física

A modelagem física é a fase final da modelagem de banco de dados e define como os dados serão armazenados fisicamente no sistema de gerenciamento de banco de dados (SGBD) escolhido. Aqui, os detalhes técnicos específicos são definidos, como tipos de dados, índices, partições, esquemas de armazenamento e regras de performance.

Principais características

  • Define tipos de dados (ex.: INT, VARCHAR, DATE) para cada coluna.
  • Define índices para otimizar as consultas de dados.
  • Inclui detalhes sobre o desempenho, como a organização dos dados em disco e regras de otimização.
  • Pode incluir esquemas de partição para grandes volumes de dados.
  • Está completamente vinculado a um SGBD específico (ex.: MySQL, PostgreSQL, Oracle).

Objetivo: Garantir que os dados sejam armazenados e acessados da maneira mais eficiente possível, considerando o SGBD escolhido e as necessidades de desempenho.
